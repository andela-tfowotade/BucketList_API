    Introduction

    Welcome to the iBucket API. This API exposes various endpoints which allows for creating, retrieving, updating and deleting bucketlists.

    On the right is an example terminal containing various code examples that shows you how to better use the API.

    On the left are navigation helpers to help you search this document or jump through to specific sections.

    • The base url for the API is https://ibucket.herokuapp.com
    • Current version of API is version 1

    Available End Points

    Below is the list of available endpoints in the BucketList API. Some end points are not available publicly and hence, can only be accessed when you sign up and log in.

    NOTE: The API is currently at version 1, so ensure you prefix each endpoint with “api/v1/” to access them.
    End Point Functionality Public Access
    GET /api/v1/ Welcome user TRUE
    POST /api/v1/auth/create_user Create a new user TRUE
    POST /api/v1/auth/login Logs a user in TRUE
    DELETE /api/v1/auth/logout Logs a user out FALSE
    GET /api/v1/bucketlists List all the created bucket lists FALSE
    POST /api/v1/bucketlists Create a new bucket list FALSE
    GET /bucketlists/:id Get single bucket list FALSE
    PUT /bucketlists/:id Update this bucket list FALSE
    DELETE /bucketlists/:id Delete this single bucket list FALSE
    GET /bucketlists/:id/items Lists all items in the single bucket list. FALSE
    POST /bucketlists/:id/items Creates a new item in the bucket list FALSE
    PUT /bucketlists/:id/items/:item_id Updates a bucket list item FALSE
    DELETE /bucketlists/:id/items/:item_id Deletes an item in a bucket list FALSE

    Authentication

    Creating an Account

    Request Format

    # The backward slash helps to write the command in multiple lines
curl -X POST -d "email=johndoe@gmail.com&password=password& \
                password_confirmation=password"
                https://ibucket.herokuapp.com/api/v1/auth/create_user

    The above returns a JSON response in this format:

        {
      "id": 11,
      "email": "johndoe@gmail.com"
    }

    You create an account by making a post request to the api/v1/auth/create_user endpoint and passing in valid parameters as part of the request.

    HTTP Request

    POST https://ibucket.herokuapp.com/api/v1/auth/create_user

    Query Parameters

    Parameter Description
    email a valid email in the format xxx@domain.com
    password a valid string with a minium of 7 characters
    password_confirmation same word as that of password

    Logging in

    Request Format

    # The backward slash helps to write the command in multiple lines
curl -X POST -d "email=johndoe@gmail.com&password=password" \
    https://ibucket.herokuapp.com/api/v1/auth/login

    The above returns a JSON response in this format:

        {
      "auth_token": "sample-token",
      "user": {
        "id": 11,
        "email": "johndoe@gmail.com"
      }
    }

    You log in to your account by making a post request to the api/v1/auth/login endpoint and passing in valid parameters as part of the request.

    Upon successful login, an api token is generated for your account and this token has to be passed (using headers) along with every request when you want to make bucketlist operations.

    HTTP Request

    POST https://ibucket.herokuapp.com/api/v1/auth/login

    Query Parameters

    Parameter Description
    email email used when creating account
    password password supplied when creating account

    Logging out

    Request Format

    # The backward slash helps to write the command in multiple lines
curl -X GET https://ibucket.herokuapp.com/api/v1/auth/logout \
    -H "token: add-your-token-here"

    The above returns a JSON response in this format:

        {
      "message": "Log out successful!"
    }
    Accessing this API endpoint requires an authorized user with a valid API token.

    You log out of your account by making a GET request to the /api/v1/auth/logout endpoint and passing in your api token in the authorization header.

    HTTP Request

    GET https://ibucket.herokuapp.com/api/v1/auth/logout

    Bucket List

    Creating a bucketlist

    Request Format

    # The backward slash helps to write the command in multiple lines
curl -X POST -d "name=bucket1&created_by=John" \
    https://ibucket.herokuapp.com/api/v1/bucketlists \
    -H "token: add-your-token-here"

    The above returns a JSON response in this format:

        {
      "id": 1,
      "name": "bucket1",
      "created_by": "John",
      "items": []
    }
    Accessing this API endpoint requires an authorized user with a valid API token.

    A new bucketlist is created by making a POST request to the /api/v1/bucketlists endpoint and passing in valid parameters as part of the request.

    HTTP Request

    POST https://ibucket.herokuapp.com/api/v1/bucketlists

    Query Parameters

    Parameter Description
    name the name of the bucket list
    created_by the owner of the bucket list

    Fetching all bucket lists

    Request Format

    # The backward slash helps to write the command in multiple lines
curl -X GET https://ibucket.herokuapp.com/api/v1/bucketlists \
    -H "token: add-token-here"

    The above returns a JSON response in this format:

        {
      "id": 11,
      "name": "bucket1",
      "created_by": "John",
      "items": []
    }
    Accessing this API endpoint requires an authorized user with a valid API token.

    This endpoint is used to retrieve all bucketlists created by a particular user.

    HTTP Request

    GET https://ibucket.herokuapp.com/api/v1/bucketlists

    Fetching a single bucket list

    Request Format

    # The backward slash helps to write the command in multiple lines

    GET https://ibucket.herokuapp.com/api/v1/bucketlists/1

    Pagination

    Request Format

    # The backward slash helps to write the command in multiple lines
curl -X GET -d "page=2&limit=20" https://ibucket.herokuapp.com \
    /api/v1/bucketlists -H "token: add-token-here"

    The above returns a JSON response in this format:

        "20 bucket list records belonging to the logged in user."
    Accessing this API endpoint requires an authorized user with a valid API token.

    This endpoint is used to request for a paginated list of bucket lists. It defaults to 20 bucket lists per page when limit is not set and defaults to 100 bucket lists per page when limit exceeds 100.

    HTTP Request

    GET https://ibucket.herokuapp.com/api/v1/bucketlists

    Query Parameters

    Parameter Description
    page page number to start displaying bucket list records from
    limit number of bucket lists to display per page

    HTTP Request

    Query Parameters

    q the name of the bucket list(s) to search for
